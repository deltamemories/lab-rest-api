## **📌 Описание задачи**

Разработайте **веб\-приложение с REST API** для управления сущностями вашей темы (см. вариант темы ниже). Приложение должно поддерживать регистрацию и авторизацию пользователей, CRUD-операции над основной сущностью, кэширование, асинхронные задачи и тестирование.

Приложение должно быть упаковано в **Docker** и запускаться одной командой.

---

## **📦 Общие требования**

* Язык: **PHP 8+**  
* Фреймворк: **Symfony 6+**  
* База данных: **MySQL (InnoDB)**  
* ORM: **Doctrine**  
* Контейнеризация: **Docker \+ docker-compose**  
* Кэш: **Redis**  
* Очередь: **RabbitMQ**  
* Тестирование: **PHPUnit**  
* CI/CD: **GitHub Actions**  
* Документация API: **Swagger/OpenAPI**  
---

  ## **🔑 Ключевые моменты**

Везде, где указано «Основная сущность / Вариант», нужно использовать **сущности вашего варианта из списка 25 тем** (например, Task, Post, Recipe, Movie и т.д.)

---

### **1\. Пользователи**

* Регистрация (`email`, `password`)  
* Авторизация (JWT или стандартная Symfony security)  
* Один пользователь может управлять только своими сущностями («Вариант основная сущность»)  
---

  ### **2\. Основная сущность вашего варианта**

* Поля (пример универсально):  
  * `id` (PK)  
  * `title` (строка)  
  * `description` (текст)  
  * `status` (строка/enum) – если применимо  
  * `created_at`, `updated_at`  
  * `user_id` (FK на User)  
* CRUD-операции через API  
* Возможность фильтрации по статусу или другому ключевому полю  
---

  ### **3\. REST API**

Обязательные эндпоинты:

| Метод | Эндпоинт | Описание |
| ----- | ----- | ----- |
| POST | /api/register | регистрация |
| POST | /api/login | авторизация |
| GET | /api/{entity} | список ваших сущностей |
| POST | /api/{entity} | создание сущности |
| PUT | /api/{entity}/{id} | обновление сущности |
| DELETE | /api/{entity}/{id} | удаление сущности |

`{entity}` — название вашей основной сущности (например, `tasks`, `posts`, `recipes`)

Все ответы должны быть в формате JSON с корректными HTTP-статусами и сообщениями об ошибках.

---

### **4\. Redis**

* Использовать для:  
  * хранения пользовательских сессий  
  * кэширования GET-запроса списка сущностей

---

  ### **5\. RabbitMQ**

* После регистрации пользователя или создания основной сущности отправлять сообщение в очередь  
* Консольный воркер обрабатывает очередь и пишет лог (например, «Пользователь зарегистрирован» или «Сущность создана»)  
---

  ### **6\. Тестирование**

* Юнит-тесты для сервисов (валидация, бизнес-логика)  
* Интеграционные тесты для CRUD API  
* Code coverage ≥ 60%  
---

  ### **7\. CI/CD**

* GitHub Actions workflow:  
  * запуск тестов при push  
  * проверка code coverage

---

  ### **8\. Документация**

* OpenAPI/Swagger для API  
* README.md с инструкцией:  
  * как собрать и запустить проект (`docker-compose up`)  
  * как выполнить миграции  
  * как запустить тесты  
  * примеры запросов к API

---

  ## **✅ Критерии оценки**

* Архитектура и качество кода — 30%  
* Реализация API и бизнес-логики — 30%  
* Интеграция Redis \+ RabbitMQ — 15%  
* Тестирование \+ CI/CD — 15%  
* Документация и README — 10%

Для получения максимального балла студент должен использовать **вариант своей темы** везде, где это указано.
